科学数据格式

Protobuf

谷歌开发的，可以根据自定义数据格式，跨平台进行序列化与反序列化

缺点

  • 不适合M级以上的数据

Protocol buffer messages are less than maximally efficient in both size and speed for many scientific and engineering uses that involve large, multi-dimensional arrays of floating point numbers. For these applications, FITS and similar formats have less overhead.

简而言之，对于大量、多维的科学运算的数据，存储与传输使用Protocolbuf开销大，官方文档给出了另一个格式的解决方案------FITS

  • 速度慢

Cap'n Proto

Protobuf同一个作者开发，对标Protobuf

优点

  • 和Protobuf比较速度快

Cap'n Proto is an insanely fast data interchange format and capability-based RPC system. Think JSON, except binary. Or think Protocol Buffers, except faster. In fact, in benchmarks, Cap'n Proto is INFINITY TIMES faster than Protocol Buffers.

夸了一顿Cap‘n Proto的速度，Cap'n Proto 编码后的数据格式直接存放在内存，数据结构跟在内存里面的布局保持一致，所以可以直接将编码好的结构根据字节存放到硬盘，或者通过网络传输。

缺点

  • 如果根据内存布局直接讲数据存入硬盘，是否意味着该工具不是跨平台的？

NO! The encoding is defined byte-for-byte independent of any platform. However, it is designed to be efficiently manipulated on common modern CPUs. Data is arranged like a compiler would arrange a struct – with fixed widths, fixed offsets, and proper alignment. Variable-sized elements are embedded as pointers. Pointers are offset-based rather than absolute so that messages are position-independent. Integers use little-endian byte order because most CPUs are little-endian, and even big-endian CPUs usually have instructions for reading little-endian data.

虽然开头一个大大的NO，但作者还是说这个工具是针对常见的现代CPU设计的，作者解释道，大部分现代CPU都有小端读取数据功能

FlatBuffers

为对性能要求高的应用开发的序列化库

优点

  • 跨平台

FlatBuffers is an efficient cross platform serialization library for C++, C#, C, Go, Java, Kotlin, JavaScript, Lobster, Lua, TypeScript, PHP, Python, Rust and Swift. It was originally created at Google for game development and other performance-critical applications.

FITS

缺点

  • Advanced Scientific Data Format (ASDF) is a proposed replacement to the FITS standard for astronomical images and other scientific data.

    简而言之，有一种新的格式ASDF被提出，用于取代FITS数据格式

ASDF

Advanced Scientific Data Format

Github主页

优点

  • A hierarchical, human-readable metadata format (implemented using YAML)
  • Numerical arrays are stored as binary data blocks which can be memory mapped. Data blocks can optionally be compressed.
  • The structure of the data can be automatically validated using schemas (implemented using JSON Schema)
  • Native Python data types (numerical types, strings, dicts, lists) are serialized automatically
  • ASDF can be extended to serialize custom data types

简而言之，ASDF类型文件的数组可以用一种二进制数据块的形式存储，并可以选择性压缩

缺点

The Advanced Scientific Data Format (ASDF) is a next-generation interchange format for scientific data. This package contains the Python implementation of the ASDF Standard. More information on the ASDF Standard itself can be found here.

Python实现，社区不活跃
